Introducción

El lenguaje de programación Go es un proyecto de código abierto para hacer programadores más productivos.

Go es expresivo, conciso, limpio y eficiente. Sus mecanismos de concurrencia facilitan la escritura de programas consiguiendo lo mejor de máquinas multinúcleo y de la red, mientras su novel sistema de tipos permite la construcción de programas flexibles y modulares.

Go compila rápidamente a código máquina aún con la comodidad de la recolección de basura y el poder de reflexión en tiempo de ejecución. Es un lenguaje tipado estáticamente, compilado y por lo tanto rápido, que se siente como un lenguaje interpretado y tipado dinámicamente.

Características
  • Go usa una sintaxis parecida a C, por lo que los programadores que hayan usado dicho lenguaje se sienten muy cómodos con él.
  • Go usa tipado estático (statically typed) y es tan eficiente como C.
  • Go tiene muchas de las características y facilidad de lenguajes dinámicos como Python
  • Aún siendo un lenguaje diseñado para la programación de sistemas, provee de un recolector de basura, reflexión y otras capacidades de alto nivel que lo convierten en un lenguaje muy potente.
  • Go admite el paradigma de programación orientada a objetos, pero a diferencia de los lenguajes de programación más populares no dispone de herencia de tipos y tampoco de palabras clave que denoten claramente que soporta este paradigma. Otro detalle que puede resultar confuso es que la definición de un tipo ("clase") se realiza por medio de declaraciones separadas (interfaces, structs, embedded values). Go permite el uso de delegación (a través de embedded values) y polimorfismo (por medio de interfaces).
    • Hola Mundo

    Para comenzar a escribir código Go, abra un editor de texto y escriba su primer código Go "Hola mundo":

    package main
    import "fmt"
    func main() {
    fmt.Printf("hola, mundo\n")
    }
    Variables

    Las variables en GO tienen algunas características dentro de las que podemos mencionar algunas:

  • Los tipos de datos son estáticos, es decir que una variable no puede cambiar su tipo de dato durante su ciclo de vida.
  • Los tipos de datos básicos en GO se pueden encontrar en el siguiente link: https://golang.org/ref/spec#Types
  • Los valores iniciales, de las variables a las cuales no se les ha asignado ningún valor, son asignados a su valor “zero”. Es decir: Los tipos numéricos inician en 0, los tipos string inician en vacio “”, los tipos boolean inician en falso, los campos de los tipos struct inician en su valor “zero”, los demás tipos inician en nil (nulo).
    • Declaración de Variables

    La estructura de la declaración de una variable es la siguiente:

    var nombre tipo = expresión

    Inicia con la palabra reservada var, luego el nombre de la variable, luego el tipo de dato que vamos a utilizar, seguido del operador de asignación “=” y por último la expresión a asignar a la variable. Sin embargo, la expresión es opcional y el tipo de dato es opcional si se declara la expresión, pero no se pueden omitir el tipo y la expresión al tiempo.

    También es posible asignar un grupo de variables con una sola palabra reservada var, y este grupo debe ser encerrado entre paréntesis. A esta declaración se le conoce como declaración múltiple de variables.

    var (
    nombre tipo
    nombre2 tipo
    nombre3 tipo
    )     También se puede asignandole valores directamente var (
    a = 1
    b = “hola”
    c = true
    )
    Constantes
    Go también tiene soporte para constantes.

    Las constantes son básicamente variables cuyos valores no se pueden cambiar más adelante. Se crean de la misma manera que creas variables, pero en lugar de usar la palabra reservada var usamos la palabra reservada const:

    paquete main
    import "fmt"
    func main () {
    const x string = "Hello World"
    fmt.Println (x)
    }
    Tipos

    Go tiene varios tipos incorporados, incluidos los numéricos ( byte , int64 , float32 , etc.), booleanos y cadenas de caracteres ( cadena ). Las cuerdas son inmutables; Los operadores integrados y las palabras clave (en lugar de las funciones) proporcionan concatenación, comparación y codificación / decodificación UTF-8 .Los tipos de estrcutura se pueden definir con la palabra clave struct .

    Sistema de Paquetes

    En el sistema de paquetes de Go, cada paquete tiene una ruta (por ejemplo, "compress/bzip2" o "golang.org/x/net/html") y un nombre (por ejemplo, bzip2o html) . Las referencias a las definiciones de otros paquetes siempre deben ir precedidas con el nombre del otro paquete, y solo los nombres en mayúsculas de otros paquetes son accesibles: io.Readeres público pero bzip2.reader no lo es. El comando go get puede recuperar paquetes almacenados en un repositorio remoto como GitHub , y se recomienda a los desarrolladores que desarrollen paquetes dentro de una ruta base correspondiente a un repositorio de origen (como github.com/user_name/package_name) para reducir la probabilidad de colisión de nombres con futuras adiciones a la biblioteca estándar u otras bibliotecas externas. Existen propuestas para introducir una solución de administración de paquetes adecuada para Go similar al sistema de carga de Rust o al sistema npm de Node.
